Types, state machines residing in a certain state, which transform one-sided into a next Type. The Transaction object of the previous state is stored for re-use.

PushdownFrom is designed to be used together with PullupFrom because one part of it's functionality is to store the previous state's Transaction onto a stack. Generally each [PushDown] must be followed with a matching Pullup operation to correctly push onto and pop from the stackstorage.

A state machine is said to pushdown from A into B when the current state is A, a Transaction object for state B is provided and the following transition is valid [A -> B].
